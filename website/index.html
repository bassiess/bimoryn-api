Quick Start

1 — Health check (no auth required)

bash 
curl https://bimoryn-api.onrender.com/health

# Expected response:
{"status": "ok", "version": "0.1.0", "rules_loaded": 35}

2 — List all validation rules

bash 
curl https://bimoryn-api.onrender.com/rules \
  -H "Authorization: Bearer bim_uLNkHebLnRbIXnJYi8LFeMBt"

3 — Validate an IFC model

bash 
curl -X POST https://bimoryn-api.onrender.com/validate \
  -H "Authorization: Bearer bim_uLNkHebLnRbIXnJYi8LFeMBt" \
  -F "file=@your-model.ifc"

# Pass/fail gate: summary.errors === 0 means model is handover-ready

Getting Started in 3 Steps
  1. Verify the API is live
    Run the health check above. If it times out the first time, wait 30s and retry — the service is waking up.
  2. Upload your first IFC model
    Use the POST /validate endpoint with any IFC4 or IFC2x3 file. The response includes per-rule issues with element references.
  3. Review results & share feedback
    Check summary.errors for the pass/fail signal. Email us any questions or rule suggestions — we iterate fast.
